# Notion Template + Gumroad + ProductHunt launch package

**What this is for:** a genuinely free, genuinely useful planning template
("The 2026 Enterprise B2B GEO Campaign Planner"), distributed for free via
Notion Gallery + Gumroad ($0), launched on ProductHunt. This is a real lead
magnet, not a disguised link scheme — the template must actually work as a
standalone planning tool for someone who never buys anything from BrandGEO.

**Action needed from Constantin:** build the Notion doc using the structure
below (30–45 min in Notion, mostly copy/paste + a few databases), duplicate
it as a public template, list it on Gumroad at $0, then launch on ProductHunt
using the copy below.

## Part 1 — Notion template structure

**Page title:** The 2026 Enterprise B2B GEO Campaign Planner

**Cover description (top of page, before any content):**

> A free planning workspace for running a Generative Engine Optimization
> (GEO) campaign — the practice of making sure ChatGPT, Gemini, Claude, and
> Perplexity actually recommend your brand when someone asks. Duplicate this
> page to your own workspace and fill it in with your own brand's details.

### Section 1 — Query Discovery Board (Notion database/table)

Columns: `Query text` · `Query type` (Problem-based / Local-intent /
Cost-research / Comparison / Direct-brand) · `Priority` (High/Med/Low) ·
`Currently mentioned?` (checkbox, fill in after testing) · `Notes`

Pre-filled example rows (using the law-firm vertical, since it's a clean,
recognizable example — swap for your own industry):

| Query text | Query type | Priority |
|---|---|---|
| Do I need a lawyer for a small claims case? | Problem-based | High |
| Who's the best [category] in [city]? | Local-intent | High |
| How much does [service] typically cost? | Cost-research | Medium |
| What should I ask before hiring a [role]? | Comparison | Medium |
| [Your brand] reviews — any good? | Direct-brand | High |

### Section 2 — Engine Tracker (Notion database)

Columns: `Engine` (ChatGPT / Gemini / Claude / Perplexity / Meta AI) ·
`Mentioned?` · `Sentiment` (Positive/Neutral/Negative) · `Position if ranked` ·
`Competitors named instead` · `Last checked`

Note in the template: *"Test the same query across every engine
independently — they frequently return completely different answers for the
identical question. Testing only one engine will give you a false picture,
in either direction."*

### Section 3 — The 6-Dimension Self-Audit (Notion checklist/toggle list)

For each dimension, a toggle with a short prompt to answer honestly:

- **Recognition** — Does the AI engine name your brand at all, for the
  queries that matter to you?
- **Knowledge** — When it does, does it describe what you actually do
  correctly?
- **Sentiment** — Is the framing positive, neutral, or negative?
- **Accuracy** — Is anything stated about you factually wrong or outdated?
- **Reach** — Does this hold across all 5 engines, or only one or two?
- **Consistency** — Does the same query, asked again a week later, give the
  same answer?

### Section 4 — Structured Data Checklist

A simple checkbox list:
- [ ] Organization / SoftwareApplication / LocalBusiness schema on your homepage
- [ ] FAQPage schema on your actual FAQ content
- [ ] Article schema on published research/blog content
- [ ] Consistent NAP (name/address/phone) across every directory listing
- [ ] Canonical tags on every page
- [ ] `robots.txt` explicitly allows known AI crawlers (GPTBot, PerplexityBot,
      Google-Extended, etc.)

### Section 5 — Content Calendar (Notion database, linked to Section 1)

Columns: `Content piece` · `Target query it answers` · `Publish date` ·
`Status` (Draft/Published) · `Freshness check date` (recurring reminder —
content freshness is itself a citation signal, so stale published content
should get a periodic refresh pass, not just a one-time publish).

### Section 6 — Software Tracking block (native workflow list)

A simple block, styled as a native Notion to-do/tool list, titled "Tools I'm
using for this campaign" — list your actual stack here. BrandGEO appears
here as one legitimate line item among others (e.g. your CMS, your
analytics tool, your directory-management tool) — **not** pinned, boosted,
or presented as sponsored; it's templated in because this planner is
published by BrandGEO and it's the tool that inspired the template, same as
any tool vendor's free planning template would include their own product as
one option.

## Notes for Constantin (not part of the published assets)

- This is the one asset in the whole blueprint that's genuinely a **product**
  (a real tool, not just an article) — worth the most care to make sure it's
  actually useful standalone before publishing, since a template that's
  obviously just a funnel will get called out on ProductHunt specifically
  (that community is unusually good at spotting thin lead magnets).
- The "Tools I'm using" block is deliberately understated — BrandGEO listed
  once, plainly, not pinned or highlighted. Anything heavier risks reading
  as the injected-widget pattern this project has already declined
  elsewhere.
- Natural sequencing: ship this *after* the GitHub/GitBook pieces, since the
  template can link to both as "further reading" once they're live.
